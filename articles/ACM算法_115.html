1 问题描述

给定序列的子序列是给定序列中遗漏了某些元素（可能没有）的序列。给定序列X = <x1, x2，…， xm>另一个序列Z = <z1, z2，…， zk>是X的子序列，如果存在严格递增的序列<i1, i2，…，如>的X指标，使得对于所有j = 1,2，…，k, xij = zj。例如，Z = <a, b, f, c>是X = <a, b, c, f, b, c>的子序列，索引序列< 1,2,4,6 >。给定两个序列X和Y，问题是找到X和Y的最大长度公共子序列的长度。

6 问题分析

问题背景描述

本题要求找到两个字符串的最长公共子序列（Longest Common Subsequence, LCS）的长度。一个子序列是从原字符串中移除某些字符（可以是零个或多个）后剩下字符按原顺序排列而成的字符串。例如，字符串"abcfbc"的子序列可以是"abc"、"bcb"等。

• 输入：多个测试用例，每个测试用例由两个字符串组成，之间由任意数量的空格分隔。

• 输出：每个测试用例输出两个字符串的LCS长度，每个结果占一行。

符号说明

1. s1, s2：两个输入字符串。

2. len1, len2：字符串 s1 和 s2 的长度。

3. dp[i][j]：表示字符串 s1[0..i-1] 和 s2[0..j-1] 的最长公共子序列长度。

解题思路和方法

1. 动态规划模型

本问题是经典的动态规划问题，求解 LCS 采用二维动态规划表格 dp：

• 状态转移方程：

• 当当前字符匹配时：dp[i][j] = dp[i-1][j-1] + 1

• 当当前字符不匹配时：dp[i][j] = \max(dp[i-1][j], dp[i][j-1])

• 边界条件：

• 初始状态：若任意字符串为空，则 dp[i][0] = dp[0][j] = 0。

2. 算法步骤

1. 初始化二维数组 dp：

• dp[i][j] 存储当前子问题的结果。

• 初始值为 0，表示空字符串的 LCS 长度。

2. 递推填表：

• 遍历两个字符串的所有字符组合，更新 dp[i][j] 的值。

3. 输出结果：

• dp[len1][len2] 为最终答案，表示两个字符串的 LCS 长度。

数据结构

1. 动态规划表 dp：

• 使用二维数组存储状态转移结果，大小为 (len1+1) x (len2+1)，时间复杂度和空间复杂度均为O(len1 \times len2)。

2. 辅助变量：

• 使用字符串长度 len1 和 len2 避免多次调用 s1.length() 和 s2.length()。

10 解题备注

经验与心得：

1. 预处理的重要性：

- 对于涉及动态规划或组合数学的问题，预先计算和存储一些中间结果（如阶乘、排列数等）可以显著提高后续计算的效率。

- 在本题中，通过预计算`nums`数组，减少了重复计算的时间复杂度。

2. 理解组合数学公式：

- 掌握基本的组合数学公式（如排列数、组合数）对于解决类似问题至关重要。

- 本题中的关键是理解并正确应用组合数公式`C(n, m) = n! / (m! * (n - m)!)`。

3. 注意输入输出格式：

- 确保输入输出格式符合要求，特别是在处理多个测试用例时，要正确读取和输出每个测试用例的结果。

4. 边界条件处理：

- 在处理边界条件时要小心，特别是当`N`或`M`接近极限值时，确保程序不会出错或超时。

归纳的规律与规则：

1. 组合数的性质：

- 组合数具有对称性，即`C(n, m) = C(n, n-m)`。

- 当`m > n/2`时，使用`C(n, m) = C(n, n-m)`可以减少计算量。

2. 排列数的计算：

- 排列数可以通过逐步累乘的方式计算，即从`n`开始逐步乘以每一个小于`n`的整数。

解题中注意事项

1. 数组越界：

- 在使用数组时，确保访问的索引在有效范围内，避免数组越界错误。

2. 数据类型：

- 由于组合数可能非常大，使用`long long`类型来存储结果以防止溢出。

3. 输入输出格式：

- 确保输入输出格式正确，特别是在处理多个测试用例时，要注意换行符的位置。

4. 时间复杂度：

- 虽然本题的时间复杂度较高，但由于题目限制了`N`的最大值为20，因此可以在合理的时间内完成计算。然而，对于更大的`N`值，需要考虑更高效的算法。

5. 代码可读性：

- 保持代码整洁和模块化，有助于提高可读性和可维护性。例如，可以将组合数计算封装成一个函数。
